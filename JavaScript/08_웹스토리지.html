Web Storage

A storage space inside the browser where data is stored as a Key-Value pairs
- localStorage: Permanent Storage (Data is maintained even if the browser/tab is closed)
- sessionStorage: Tab-level Storage (Data is deleted when the tab is closed)
- Major Methods: setItem(Key, Value), getItem(Key, Value), removeItem(Key), clear()
- [Note] Web Storage can only store data in STRING format

브라우저 내부에 데이터를 키-값 쌍으로 저장하는 공간
로컬 스토리지: 영구 저장(브라우저나 탭을 닫아도 데이터가 유지됨)
세션 스토리지: 탭 단위 저장(해당 탭을 닫으면 데이터가 삭제됨.)
주요 메서드: setItem(k, v), getItem(k), remove(k), clear()
참고: 웹 스토리지는 "문자열" 형태의 데이터만 저장할 수 있음

localStorage

Used for data that needs to be retained even if the browser closed or reopened
- User settings: dark mode/light mode, language preferences, etc
- Guest shopping cart: Temporarily storing shopping list data before login
- "Do not show agian today" option: Saving popup expiration time and status

브라우저를 닫거나 다시 열어도 유지되어야 하는 데이터에 사용
- 사용자 설정: 다크모드/라이트모드, 언어설정 등
- 비회원 장바구니: 로그인 전 장바구니 데이터를 임시보관
- 오늘 하루 이 창 보지 않기: 팝업창 만료 시간 및 상태 저장

sessionStorage

Used when data should not remain permanently, or to store temporarily while the current tab is open
- Form auto-save: Retain typed content even if the page is accidently reloaded while writing
- Preserving list state: Restore scroll position and filter conditons when returning from the detail page

데이터가 영구적으로 남으면 안 되거나, 현재 탭이 유지되는 동안만 일시적으로 보관할 때 사용
- 작성 폼 백업: 게시글 작성 중 실수로 새로고침을 해도 작성 중인 내용 유지
- 목록 상태 유지: 상품 목록 스크롤 위치 및 필터 조건을 유지하면서 상세 페이지에서 돌아왔을 때 복원
